# قواعد صارمة للوكيل العامل داخل هذا المشروع

1. كل ملف agent يصدّر instance جاهز (export const x = new Class()) — لا تستورد الـ class أبدًا، استورد الـ instance فقط.
2. قبل أي تعديل: نفّذ `grep -rn "export const" agents/` لمعرفة كل الـ singletons المسجلة، وتجنّب كسرها.
3. عند ظهور أي خطأ (TypeError, SyntaxError, استيراد فاشل، جدول مفقود): لا تتوقف لتسأل — افحص السبب الجذري، طبّق الحل المنطقي البديل، وأكمل.
4. بعد أي تعديل: شغّل `node --check` على كل الملفات المعدّلة قبل الانتقال للخطوة التالية.
5. بعد أي إصلاح ناجح: نفّذ commit + push تلقائيًا بوصف دقيق للتغيير.
6. لا تكرر هذا النوع من الأخطاء مستقبلاً: عند إضافة agent جديد، تحقّق أولاً من نمط التصدير في الملفات المشابهة وقلّده.
7. الهدف الدائم: brain_memory يجب أن يستمر بالنمو، الوكلاء 94 يجب أن تعمل، الحوكمة تراقب القرارات — أي خلل في هذا المسار له أولوية الإصلاح القصوى.
