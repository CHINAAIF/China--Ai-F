# قواعد صارمة للوكيل العامل داخل هذا المشروع

1. كل ملف agent يصدّر instance جاهز (export const x = new Class()) — لا تستورد الـ class أبدًا، استورد الـ instance فقط.
2. قبل أي تعديل: نفّذ `grep -rn "export const" agents/` لمعرفة كل الـ singletons المسجلة، وتجنّب كسرها.
3. عند ظهور أي خطأ (TypeError, SyntaxError, استيراد فاشل، جدول مفقود): لا تتوقف لتسأل — افحص السبب الجذري، طبّق الحل المنطقي البديل، وأكمل.
4. بعد أي تعديل: شغّل `node --check` على كل الملفات المعدّلة قبل الانتقال للخطوة التالية.
5. بعد أي إصلاح ناجح: نفّذ commit + push تلقائيًا بوصف دقيق للتغيير.
6. لا تكرر هذا النوع من الأخطاء مستقبلاً: عند إضافة agent جديد، تحقّق أولاً من نمط التصدير في الملفات المشابهة وقلّده.
7. الهدف الدائم: brain_memory يجب أن يستمر بالنمو، الوكلاء 94 يجب أن تعمل، الحوكمة تراقب القرارات — أي خلل في هذا المسار له أولوية الإصلاح القصوى.

8. قبل كتابة أي INSERT: نفّذ فحص schema كامل للجدول (الأعمدة، NOT NULL، القيم الافتراضية) — لا تخمّن الأعمدة المطلوبة.
9. لا تضع سلسلة خطوات INSERT متتابعة داخل try واحد — كل خطوة لها try/catch مستقل، بحيث فشل خطوة لا يوقف ما بعدها.
10. اطبع رسالة الخطأ كاملة عند الفشل، لا تقصّها — التشخيص يحتاج النص الكامل.

11. قبل أي INSERT في عمود به CHECK constraint (مثل brain_memory.memory_type): اجلب القيم المسموحة فعليًا عبر pg_constraint، ولا تدرج قيمة خام أبدًا — استخدم خريطة تحويل ثابتة.
12. القيم المسموحة لـ brain_memory.memory_type: pricing_pattern, vendor_behavior, market_signal, content_insight, agent_performance, security_pattern, user_behavior, prediction_result.

13. لديك صلاحية تلقائية كاملة داخل هذه البيئة: افحص أي ملف، مسار، جدول، أو سجل تراه أمامك، وصححه فورًا دون انتظار تأكيد من المستخدم.
14. قبل أي INSERT/UPDATE: اقرأ db-schema-map.json أولًا (يحوي كل الأعمدة، القيود، NOT NULL، CHECK، Foreign Keys لكل القاعدة) — لا تخمّن القيم أو الأعمدة المطلوبة أبدًا.
15. إن وجدت خطأ، ناقصًا، أو مسارًا أفضل تقنيًا لم يُطلب منك صراحة: نفّذه مباشرة، ووثّقه في تعليق commit واضح، واستمر للخطوة التالية دون توقف.
16. لا تنتظر توجيهًا إضافيًا إلا إذا واجهت غموضًا حقيقيًا يهدد سلامة البيانات (مثل حذف جماعي أو تعارض منطقي لا يوجد له حل واضح من السياق).
